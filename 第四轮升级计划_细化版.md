## 2. P2 级：代码质量与逻辑修复（必做）

### P2-1：CORS 白名单配置

**当前状态**：
- `server/index.js` 第 127 行：`app.use(cors({` 已有配置，需确认是否有白名单
- 需检查是否限制了来源域名

**问题描述**：
如果 CORS 完全开放（`cors()` 无参数），任何域名均可跨域请求接口，存在安全隐患。

**修改方案**：
1. 在 `server/index.js` 中配置来源白名单
2. 从环境变量 `ALLOWED_ORIGINS` 读取（逗号分隔）
3. 默认包含开发地址：`http://localhost:3000`, `http://localhost:3002`, `http://localhost:5173`
4. 生产环境追加：`https://video.qiziwenhua.top`, `https://qiziwenhua.top`
5. 支持通配子域名（如 `*.qiziwenhua.top`）

**技术细节**：
```javascript
const allowedOrigins = (process.env.ALLOWED_ORIGINS || '').split(',').map(s => s.trim()).filter(Boolean);
const defaultOrigins = [
  'http://localhost:3000',
  'http://localhost:3002', 
  'http://localhost:5173',
  'https://video.qiziwenhua.top',
  'https://qiziwenhua.top'
];
const origins = [...new Set([...defaultOrigins, ...allowedOrigins])];

app.use(cors({
  origin: function(origin, callback) {
    // 允许无 origin 的请求（如 Postman、移动端、同域）
    if (!origin) return callback(null, true);
    // 检查白名单
    if (origins.includes(origin) || origin.endsWith('.qiziwenhua.top')) {
      callback(null, true);
    } else {
      callback(new Error('Not allowed by CORS'), false);
    }
  },
  credentials: true
}));
```

**修改文件**：
- `server/index.js`
- `.env.example`（添加 `ALLOWED_ORIGINS` 示例）

**验证方法**：
- 从非白名单域名发起跨域请求，确认被拒绝
- 从白名单域名请求，确认正常

### P2-2：stats 接口性能优化（COUNT(*) 替代全量查询）

**当前状态**：
- `server/index.js` 中 stats 接口使用 `getByFilter` 查出所有记录再 `.length` 计数
- 未使用 `SELECT COUNT(*)` 优化

**问题描述**：
获取统计数字时先查出所有完整记录，数据量大时性能差，浪费 IO 和内存。

**修改方案**：
1. 在 `server/database.js` 中新增 `video2Items.getStats(projectId, sceneId?)` 方法
2. 使用 `SELECT status, COUNT(*) as cnt FROM video2_items WHERE ... GROUP BY status`
3. 同样优化场次统计：`SELECT sceneId, status, COUNT(*) ... GROUP BY sceneId, status`
4. 计算 `unclassified`（sceneId IS NULL 的数量）也走 COUNT
5. 前端调用方式不变，接口返回结构保持一致

**技术细节**：
```sql
-- 主统计
SELECT status, COUNT(*) as cnt 
FROM video2_items 
WHERE projectId = ? AND deleted = 0
GROUP BY status

-- 未分类统计
SELECT COUNT(*) as cnt
FROM video2_items
WHERE projectId = ? AND deleted = 0 AND sceneId IS NULL
```

**修改文件**：
- `server/database.js`（新增 `getStats` 方法）
- `server/index.js`（stats 接口调用新方法）

**验证方法**：
- 检查接口返回数据正确性
- 对比优化前后响应时间（大数据量下应有明显提升）
- 确认各 Tab 下统计数字准确

### P2-4：ShotCard 图片错误改用 React State

**当前状态**：
- `src/components/storyboard/ShotCard.tsx` 中图片加载失败时使用 `document.createElement` 直接操作 DOM
- 绕过了 React 虚拟 DOM，可能导致状态不一致

**修改方案**：
1. 添加 `const [imgError, setImgError] = useState(false)`

3. 条件渲染：`imgError ? <占位符组件> : <img ... />`
4. 移除所有 `document.createElement`、`appendChild`、`parentNode` 等 DOM 操作
5. 占位符样式与现有设计一致（灰色背景 + ImageIcon 图标）

**技术细节**：
```tsx
const [imgError, setImgError] = useState(false);

{imgError ? (
  <div className="w-full h-full flex items-center justify-center bg-slate-800/50">
    <ImageIcon className="w-8 h-8 text-slate-500" />
  </div>
) : (
  <img
    src={url}
    onError={() => setImgError(true)}
    className="w-full h-full object-cover"
    alt=""
  />
)}
```

**修改文件**：
- `src/components/storyboard/ShotCard.tsx`

**验证方法**：

### P3-9：分镜备份/恢复功能

**优先级**：低 **难度**：中 **预计工时**：3-4 小时

**功能描述**：
- 自动备份：每天凌晨自动备份 SQLite 数据库到 `server/backups/` 目录
- 保留策略：保留最近 7 天的备份
- 手动备份：设置页面提供「立即备份」按钮
- 恢复备份：设置页面显示备份列表，可选择恢复某一备份

**技术方案**：

后端自动备份：
```javascript
// 使用 node-cron 或简单的 setInterval
const BACKUP_DIR = path.join(__dirname, 'backups');
const KEEP_DAYS = 7;

function createBackup() {
  const timestamp = new Date().toISOString().slice(0, 10);
  const backupPath = path.join(BACKUP_DIR, `video2_${timestamp}.db`);
  fs.copyFileSync(DB_PATH, backupPath);
  // 清理 7 天前的备份
  const files = fs.readdirSync(BACKUP_DIR).filter(f => f.endsWith('.db'));
  files.forEach(f => {
    const dateStr = f.match(/video2_(\d{4}-\d{2}-\d{2})\.db/)?.[1];
    if (dateStr) {
      const days = (Date.now() - new Date(dateStr).getTime()) / 86400000;
      if (days > KEEP_DAYS) fs.unlinkSync(path.join(BACKUP_DIR, f));
    }
  });
}

// 每天凌晨 3 点执行
cron.schedule('0 3 * * *', createBackup);
```

### P3-10：API 接口鉴权

**优先级**：低 **难度**：中 **预计工时**：2-3 小时

**功能描述**：
- 为设置接口（`/api/video2/settings`）添加简单鉴权
- 防止公网暴露时任意修改 API Key 等敏感配置

**技术方案**：

方案 A（简单版，HTTP Basic Auth）：
```javascript
// 从环境变量读取管理员密码
const ADMIN_TOKEN = process.env.ADMIN_TOKEN || '';

function adminAuth(req, res, next) {
  if (!ADMIN_TOKEN) return next(); // 未配置则不鉴权
  const token = req.headers['x-admin-token'];
  if (token === ADMIN_TOKEN) {
    next();
  } else {
    res.status(401).json({ error: 'Unauthorized' });
  }
}

app.get('/api/video2/settings', adminAuth, (req, res) => { ... });
app.put('/api/video2/settings', adminAuth, (req, res) => { ... });
```

方案 B（IP 白名单）：
```javascript
const ADMIN_IPS = (process.env.ADMIN_IPS || '').split(',').map(s => s.trim());

function ipWhitelist(req, res, next) {
  if (ADMIN_IPS.length === 0) return next();
  const clientIp = req.ip || req.connection.remoteAddress;
  if (ADMIN_IPS.includes(clientIp)) {
    next();
  } else {
    res.status(403).json({ error: 'Forbidden' });
  }
}
```

推荐：方案 A（Token 鉴权），更灵活，不受网络环境限制。

**修改文件**：
- `server/index.js`（鉴权中间件 + 受保护路由）
- `.env.example`（添加 `ADMIN_TOKEN` 示例）
- 前端设置页面（输入管理员 Token 才能访问设置）

**验收标准**：
- 无 Token 时访问设置接口返回 401
- 正确 Token 时正常访问
- 前端设置页面支持输入 Token 并保存到 localStorage

### P3-13：AI 任务 SSE 实时推送

**优先级**：低 **难度**：高 **预计工时**：6-8 小时

**功能描述**：
- 使用 Server-Sent Events (SSE) 替代轮询
- 任务进度变化时主动推送给前端
- 一条 SSE 连接可复用多个任务

**技术方案**：

后端：
```javascript
// 保存客户端连接
const sseClients = new Map(); // taskId -> [res1, res2, ...]

app.get('/api/video2/ai/task/:taskId/stream', (req, res) => {
  const taskId = req.params.taskId;
  res.writeHead(200, {
    'Content-Type': 'text/event-stream',
    'Cache-Control': 'no-cache',
    'Connection': 'keep-alive'
  });
  
  const clients = sseClients.get(taskId) || [];
  clients.push(res);
  sseClients.set(taskId, clients);
  
  req.on('close', () => {
    const updated = (sseClients.get(taskId) || []).filter(c => c !== res);
    sseClients.set(taskId, updated);
  });
});

// 任务进度更新时推送
function notifyTaskUpdate(taskId, data) {
  const clients = sseClients.get(taskId) || [];
  clients.forEach(res => {
    res.write(`data: ${JSON.stringify(data)}\n\n`);
  });
}
```

前端：
```tsx
const evtSource = new EventSource(`/api/video2/ai/task/${taskId}/stream`);
evtSource.onmessage = (e) => {
  const data = JSON.parse(e.data);
  setTask(prev => ({ ...prev, ...data }));
  if (data.status === 'completed' || data.status === 'failed') {
    evtSource.close();
  }
};
```

**修改文件**：
- `server/index.js`（SSE 端点 + 推送逻辑）
- `src/components/ai/AIScriptDialog.tsx`（SSE 客户端）
- `src/components/ai/AIImageGenDialog.tsx`（SSE 客户端）

**验收标准**：
- AI 任务进度实时更新（无轮询延迟）
- 多个任务可同时进行，互不干扰
- 网络断开后自动重连（EventSource 内置）
- 页面关闭后连接正确释放

## 6. 风险与注意事项

### 6.1 安全风险

| 风险项 | 影响 | 应对措施 |
|--------|------|---------|
| CORS 配置错误 | 导致正常跨域请求被拒绝 | 配置白名单时保留本地开发地址，测试通过后再部署 |
| API 鉴权 Token 泄露 | 设置接口被未授权访问 | Token 存环境变量，不在代码中硬编码；前端使用 HTTPS |

### 6.2 兼容性风险

| 风险项 | 影响 | 应对措施 |
|--------|------|---------|
| stats 接口重构 | 返回格式变化导致前端报错 | 保持返回结构与原来完全一致，仅优化内部实现 |
| SSE 浏览器兼容 | 旧浏览器不支持 EventSource | 保留轮询作为 fallback，检测不支持时自动降级 |

### 6.3 性能风险

| 风险项 | 影响 | 应对措施 |
|--------|------|---------|
| PDF 导出大项目超时 | 导出失败 | 大项目使用异步任务，完成后通知下载 |
| 备份占用磁盘空间 | 磁盘不足 | 限制保留数量（7天），监控磁盘使用率 |
